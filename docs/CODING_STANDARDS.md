# Coding Standards

This document defines the coding standards used in the Jarvis project.

These standards ensure:

- consistent code structure
- maintainable architecture
- high readability
- predictable patterns

All contributors and automated agents must follow these standards.

---

# Language

The primary language used in Jarvis is:

TypeScript

JavaScript should not be used unless absolutely necessary.

---

# Framework

Backend services are implemented using:

NestJS

Developers must follow standard NestJS architectural practices.

---

# General Principles

All code must follow these principles:

- clarity over cleverness
- explicit logic over implicit behavior
- modular design
- small focused functions

Avoid complex or overly abstract implementations.
